# YOLO OBB Workflow

This folder contains the Python-side YOLO tooling for the dart dataset. The current dataset uses OBB labels, so training and evaluation should use the OBB task, not plain detect.

## Files

- `train.py`: trains a YOLO model on the dataset and auto-generates `dart_dataset.yaml`
- `play.py`: runs prediction on images, folders, video, or streams
- `model.py`: configures local Ultralytics paths and loads the model
- `dataset.py`: builds the dataset YAML from a dataset root
- `visualize_yolo_obb.py`: draws OBB labels on images
- `refine_labels.py`: image-processing helper for rough label refinement
- `requirements.txt`: Python dependencies

## Environment

From the repository root:

```bash
cd .
UV_CACHE_DIR=.uv-cache uv venv yolo
source yolo/bin/activate
```

Install dependencies:

```bash
UV_CACHE_DIR=.uv-cache uv pip install --python yolo/bin/python -r python/yolo/requirements.txt
```

If you want the CPU build of PyTorch:

```bash
UV_CACHE_DIR=.uv-cache uv pip install --python yolo/bin/python --index-url https://download.pytorch.org/whl/cpu torch torchvision
UV_CACHE_DIR=.uv-cache uv pip install --python yolo/bin/python ultralytics
```

## Dataset Layout

Expected layout:

```text
dart_dataset/
  images/
    train/
    val/
    test/     # optional
  labels/
    train/
    val/
    test/     # optional
```

Current OBB label format:

```text
class_id x1 y1 x2 y2 x3 y3 x4 y4
```

All coordinates are normalized to `[0, 1]`.

`train.py` inspects the label format automatically. If the labels are OBB, it requires OBB weights.

## Weights

Recommended OBB weights:

- `yolo11n-obb.pt`: current default in `model.py`
- `yolov8n-obb.pt`: use this if you specifically want YOLOv8 OBB

Do not use `yolov8n.pt` with OBB labels.

## Training

YOLOv8 OBB on GPU:

```bash
cd python/yolo
python train.py \
  --data-root ../../dataset/dart_dataset \
  --classes dart \
  --weights yolov8n-obb.pt \
  --epochs 100 \
  --imgsz 640 \
  --batch 16 \
  --device 0 \
  --project runs \
  --name dart_yolov8_obb \
  --patience 30
```

Before using `--device 0`, confirm CUDA is visible:

```bash
python -c "import torch; print(torch.cuda.is_available(), torch.cuda.device_count())"
```

## Validation

Evaluate the `test` split with a trained OBB model:

```bash
yolo obb val \
  model=python/yolo/runs/dart_yolov8_obb/weights/best.pt \
  data=python/yolo/dart_dataset.yaml \
  split=test \
  device=0
```

If no GPU is available, replace `device=0` with `device=cpu`.

## Prediction

Run prediction on a folder or image:

```bash
cd python/yolo
python play.py \
  --weights runs/dart_yolov8_obb/weights/best.pt \
  --source ../../dataset/dart_dataset/images/test \
  --device 0
```

Direct Ultralytics CLI form:

```bash
yolo obb predict \
  model=python/yolo/runs/dart_yolov8_obb/weights/best.pt \
  source=dataset/dart_dataset/images/test \
  device=0
```

## Notes

- `model.py` rewrites Ultralytics settings so runs, weights, and dataset paths stay inside the project.
- `dart_dataset.yaml` is generated by `train.py` in this directory.
- If labels are wrong after image rotation, regenerate them from `annotations.xml` before training.
- OBB visualization images are only for inspection; training uses the `.txt` labels under `labels/`.
